# 프로젝트 브리프 — 진짜같은 가짜 (Fake but Real)

> 이 문서는 Claude Cowork에게 전달하기 위한 프로젝트 요구사항 정의서입니다.
> 아래의 내용을 바탕으로 웹사이트 구조 설계 및 개발을 진행해주세요.

---

## 1. 프로젝트 개요

### 프로젝트명
**진짜같은 가짜** (가칭 / 영문: *Fake but Real*)

### 한 줄 정의
"개념과 사진"이라는 두 가지 무기로 완전히 다른 세계들을 하나씩 만들어가는 크리에이티브 프로젝트.

### 핵심 아이디어
- 매번 전혀 다른 분야의 세계관을 설정하고 (e.g. 프로게임단, 격투기 팀, 레이싱 팀, 패션 브랜드…)
- 직접 찍은 사진과 치밀하게 설계된 콘셉트로 그 세계를 **진짜처럼** 구현한다.
- 각 프로젝트는 독립된 페이지로 존재하지만, 모두 **하나의 거대한 브랜드** 아래 묶인다.
- 보는 사람이 "이게 진짜야?" 하고 한 번쯤 의심하게 만드는 것이 목표.

## 2. 사이트 구조

### URL 구조
```
(루트 도메인)/                     ← 메인 허브 페이지
(루트 도메인)/odod/                ← 첫 번째 프로젝트: 가짜 LoL 프로게임단 ODOD
(루트 도메인)/project-02/          ← 두 번째 프로젝트 (미정)
(루트 도메인)/project-03/          ← 세 번째 프로젝트 (미정)
...
```

> **중요:** 각 프로젝트는 URL로 구분되는 독립 페이지(서브 디렉토리)로 존재해야 하며,
> 루트 허브에서 각 프로젝트로 진입할 수 있어야 합니다.
> 프로젝트 수는 계속 늘어날 예정이므로 구조가 확장 가능해야 합니다.

### 파일 구조 (권장)

## 3. 메인 허브 페이지 (`/index.html`)

### 역할
- 전체 프로젝트 시리즈의 **포털** 역할
- "진짜같은 가짜"라는 브랜드 아이덴티티를 강하게 표현
- 각 프로젝트로 진입할 수 있는 카드/링크 제공

### 필수 구성 요소
- 프로젝트명 및 슬로건 (전체 개념 소개, 1-2줄)
- 프로젝트 목록 카드 (썸네일 + 프로젝트명 + 짧은 설명 + 링크)
- 새 프로젝트가 추가될 때 카드가 자연스럽게 확장되는 구조

### 톤 & 무드
- 메인 허브는 각 프로젝트보다 **더 미니멀하고 차분하게**
- 프로젝트들의 혼재된 세계관을 하나의 일관된 프레임으로 묶는 느낌
- 과도한 컬러 없이, 타이포그래피와 레이아웃으로 브랜딩

## 4. 프로젝트 01 — ODOD (가짜 LoL 프로게임단)

### 팀 정보
- **팀명:** ODOD (OverDrive Outlaw Dogs)
- **컬러:** 오렌지 / 블랙 (웜 다크톤 배경)
- **리그:** LCK (한국 리그 오브 레전드 챔피언십)
- **창단:** 2022년, 서울
- **슬로건:** "We don't follow the meta. We break it."

### 로스터 구성 (총 8인)

|--------|--------|------|
| 감독 (Head Coach) | Vortex | |

> **중요:** 모든 인물 사진은 제작자가 직접 촬영하여 제공합니다.
> 사진 경로: `odod/assets/images/players/`, `odod/assets/images/staff/`
> 파일명 규칙: `{닉네임_소문자}.jpg` (e.g. `ironwall.jpg`, `vortex.jpg`)

### 페이지 구성 (이미 개발 중 / 계속 이어서)
1. **네비게이션** — 로고, 메뉴, SNS 링크, 팔로우 버튼
2. **히어로 섹션** — 팀 타이틀 + 슬로건, 선수 세로형 프레임 배너 (등장 애니메이션), 스태프 스트립
3. **시즌 스탯 바** — 순위, 전적, 승률, 월드챔피언십 출전 횟수
4. **전적 섹션 (Results)** — 최근 경기 결과 테이블
5. **일정 섹션 (Schedule)** — 예정 경기 목록
6. **미디어 갤러리 (Media)** — 사진 그리드 (직접 촬영한 사진)
7. **스폰서 (Partners)** — 파트너사 나열
8. **푸터**

### 선수 개인 프로필 팝업 (클릭 시 열림)
- **레이아웃:** 가로형 팝업 (좌 40% / 우 60%)
- **좌측:** 하이라이트 영상 (YouTube embed) + 선수 사진
- **우측 (스크롤 가능):**
  - 포지션 배지 + 닉네임 + 실명 + 국적
  - 개인정보: 나이, 생년월일, 등번호, 합류 연도, 총 경기 수
  - 시즌 스탯: KDA, 승률, CS/분, 킬/데스/어시스트
  - 주 챔피언 3개: 챔피언 이미지 + 승률 + KDA + 게임 수
  - 커리어 이력
  - 수상 이력
  - SNS 링크 (선택)

### 스태프 개인 팝업
- 풀 레이아웃 (영상 없음)
- 역할, 닉네임, 실명, 나이, 합류 연도
- 지도 철학 (한 줄 인용)
- 커리어 이력

### 어드민 페이지 (`/odod/admin/`)
- **진입 방식:** URL 파라미터 `?admin=ODOD2025` 입력 시 하단 바 노출 → 어드민 링크 클릭
- **어드민 비밀번호:** 별도 입력 (현재: `ODOD2025`)
- **수정 가능 항목:**
  - 팀 기본 정보 (이름, 슬로건, 시즌, 순위, 전적, SNS 링크)
  - 스태프 정보 (닉네임, 역할, 사진 경로, 철학, 커리어)
  - 선수 정보 (전체 필드, 스탯, 챔피언, 커리어, 수상, 하이라이트 URL)
  - 전적 (추가/삭제/수정)
  - 일정 (추가/삭제/수정)
  - 갤러리 (경로 + 캡션 관리)
  - 스폰서 (추가/삭제/수정)
- **저장 방식:** LocalStorage (같은 브라우저에서 뷰어 페이지에 반영)
- **GitHub Pages 배포 후:** 어드민에서 수정 → LocalStorage 저장 → 같은 브라우저의 뷰어에 반영

### 데이터 구조
- `odod/data/team.json` — 모든 데이터의 원본 (초기값)
- `odod/js/main.js` — `team.json` 데이터를 JS 내부에 `DEFAULT_DATA`로 내장 (CORS 이슈 방지)
- LocalStorage key: `odod_data` — 어드민 저장 시 이 키에 덮어씀
- 뷰어 로드 순서: LocalStorage 우선 → 없으면 `DEFAULT_DATA` 사용

## 6. 기술 스택 및 환경

### 기술
- **프론트엔드:** 순수 HTML / CSS / Vanilla JS (프레임워크 없음)
- **데이터:** JSON 파일 (서버 없음)
- **배포:** GitHub Pages
- **저장:** LocalStorage (어드민 수정사항 반영)

### 배포 환경
- GitHub 계정: 있음 (별도 안내 불필요)
- 배포 방법: GitHub Pages (repository → Settings → Pages)
- 도메인: GitHub 기본 도메인 사용 (`https://{username}.github.io/{repo}/`)
  또는 커스텀 도메인 추후 연결 가능

### 로컬 개발 시 주의사항
- `fetch()`를 사용한 JSON 로드는 로컬 파일 시스템에서 CORS 오류 발생
- 이 때문에 JSON 데이터를 JS 파일 내 `DEFAULT_DATA` 상수로 내장하는 방식 사용
- 로컬에서 `index.html` 더블클릭만으로 완전히 동작해야 함

## 7. 디자인 원칙

### 허브 페이지
- 미니멀, 다크, 타이포그래피 중심
- 컬러: 배경 딥 다크 / 텍스트 오프화이트
- 포인트 컬러 없음 또는 극도로 절제

### 각 프로젝트 페이지
- 프로젝트마다 **고유한 컬러 시스템** 사용
- 배경: 완전 블랙(#000) 금지 — 사유의방처럼 깊고 따뜻하게 빛이 도는 다크톤
- 오렌지, 레드 등 강한 포인트 컬러는 **극도로 절제** (버튼 하나, 라인 하나 수준)
- 사진이 가장 강하게 드러나야 하므로 UI 자체는 최대한 비워야 함
- 텍스트: 순백 금지, 따뜻한 오프화이트 계열 (`#f0ebe4` 등)

### 공통 UI 패턴
- 세로형 선수 카드 프레임 (클릭 시 팝업)
- 팝업: 가로형, 좌측 미디어 / 우측 정보
- 스크롤 애니메이션: 요소가 순차적으로 올라오며 등장 (`translateY + opacity`)
- 보더: `0.5px solid` 극도로 얇게

## 9. 인수인계 메모

- 현재 파일 패키지(`odod.zip`)에 전체 코드가 포함되어 있습니다.
- 압축 해제 후 `odod/index.html`을 브라우저에서 열면 바로 동작합니다.
- 어드민 접근: `odod/index.html?admin=ODOD2025` → 하단 바 클릭 → 비밀번호 `ODOD2025`
- 사진은 아직 placeholder 상태이며, 실제 촬영 후 해당 폴더에 넣으면 자동 반영됩니다.
- 데이터 수정은 어드민 페이지에서 하거나, `js/main.js`의 `DEFAULT_DATA` 객체를 직접 수정하면 됩니다.
